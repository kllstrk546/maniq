# Manicure Bot

Telegram-бот для запису клієнтів, адмін-керування слотами та публікації розкладу в канал.

## Функції

- Запис клієнта на вільний слот.
- Скасування свого запису.
- Перевірка підписки на канал перед записом.
- Адмін-панель: відкриття/закриття днів, додавання/видалення слотів, перегляд розкладу.
- Публікація/оновлення розкладу в канал.
- Нагадування через APScheduler (за 24 години, якщо до візиту > 24 год).

## Вимоги

- Python 3.11+ (рекомендовано).
- Telegram Bot Token від `@BotFather`.

## Встановлення

```bash
pip install -r requirements.txt
```

## Налаштування `.env`

Створіть файл `.env` у корені проєкту та вкажіть:

- `BOT_TOKEN` — токен бота.
- `ADMIN_ID` — ваш Telegram user id.
- `CHANNEL_ID` — id каналу для перевірки підписки (наприклад, `-100...` або `@username`).
- `CHANNEL_LINK` — публічне посилання каналу (наприклад, `https://t.me/your_channel`).
- `SCHEDULE_CHANNEL_ID` — id/username каналу, куди публікується розклад.

Опційно:

- `TIMEZONE` (за замовчуванням `Europe/Kiev`).
- `DB_PATH` (за замовчуванням `data/bot.sqlite3`).

## Права бота в каналах

### Канал підписки (`CHANNEL_ID`)

Щоб коректно перевіряти підписку та обробляти заявки:

- Бот має бути доданий у канал.
- Якщо канал закритий і є заявки на вступ, боту потрібні права на схвалення заявок.
- Для стабільної перевірки `getChatMember` рекомендовано надати боту адмін-права в каналі.

### Канал розкладу (`SCHEDULE_CHANNEL_ID`)

Щоб бот міг публікувати та редагувати пости розкладу:

- Бот має бути адміном каналу.
- Потрібні права на надсилання та редагування повідомлень.

## Поширені проблеми

### 1) `Bad Request: member list is inaccessible`

Причина: Telegram не надає боту доступ до списку учасників каналу.

Що зробити:

- Переконайтесь, що бот доданий у канал підписки.
- Надайте боту адмін-права в каналі.
- Перевірте, що `CHANNEL_ID` вказує на правильний канал.

### 2) `Bad Request: chat not found`

Причина: некоректний `CHANNEL_ID`/`SCHEDULE_CHANNEL_ID` або бот не перебуває в каналі.

Що зробити:

- Перевірте ID каналу (`-100...`) або `@username`.
- Перевірте, що бот доданий у канал.

### 3) Кнопка підписки відкриває канал, але перевірка не проходить

Що перевірити:

- Користувач реально вступив у канал.
- Бот має потрібні права в каналі.
- У `.env` коректні `CHANNEL_ID` і `CHANNEL_LINK`.

### 4) Адмін-панель не реагує

Що перевірити:

- Ваш `user_id` збігається з `ADMIN_ID` у `.env`.
- Бота перезапущено після змін: `python bot.py`.

## Логи

Логування налаштовано в `bot.py`:

- єдиний формат часу/рівня/логера;
- вивід у stdout;
- рівні для `aiogram` і `apscheduler`.

## Структура (коротко)

- `app/handlers` — користувацькі та адмінські обробники.
- `app/services` — бізнес-логіка (підписка, розклад, сповіщення, reminders).
- `app/database` — репозиторії та ініціалізація БД.
- `bot.py` — точка входу.
